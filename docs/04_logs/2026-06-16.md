### 커뮤니티 검색 결과 URL 동기화 (뒤로가기 버그 수정)

**문제**: 검색어·정렬·페이지 상태가 URL에 반영되지 않아, 글 상세 조회 후 뒤로가기 시 검색 결과가 초기화됨

**원인**: `CommunityView.vue`의 `keyword`, `sort`, `page` ref가 컴포넌트 내부 메모리에만 존재하고 URL 쿼리 파라미터와 무관하게 동작

**해결**:
- `ref` 초기값을 `route.query.*`에서 읽도록 변경
- `doSearch()`, `clearSearch()`, `changeSort()`, `goPage()` 실행 시 `router.replace({ query })` 호출로 URL 동기화
- 이제 `/community?keyword=제주&sort=popular&page=2` 형태로 URL이 유지되므로, 브라우저 뒤로가기 후 동일 검색 결과로 복귀

### 방문 지도 — D3 + 실제 행정구역 경계로 교체

**배경**: 기존에 손으로 그린 단순화 SVG path(`koreaMap.js`)를 사용했는데, 실제 한반도 모양과 차이가 커서 교체 결정

**도입 방식 — 사전 계산(Pre-compute) 전략**

브라우저 런타임에서 D3를 실행하는 대신, **Node.js 빌드 단계에서 SVG path를 미리 계산**해 정적 JS 파일로 저장하는 방식을 선택했다. 이유는 두 가지:
1. `d3-geo`를 브라우저에서 동적 import 시 Vue 컴포넌트 초기화 에러가 발생해 화면이 빈 채로 렌더링되는 문제가 있었음
2. 런타임에 무거운 GeoJSON 변환 연산이 불필요해져 초기 로딩 속도가 빠름

**데이터 흐름**

```
Highcharts 공개 TopoJSON (17개 시도 경계)
  → topojson-client.feature()  ← TopoJSON → GeoJSON FeatureCollection 변환
  → d3-geo geoMercator().fitSize([500, 620], geoJson)  ← 좌표 투영 설정
  → geoPath().projection(proj)  ← SVG path 생성기
  → 각 feature마다 pathGen(f) → SVG path 문자열
  → pathGen.centroid(f) → 레이블 중심 좌표 [x, y]
  → koreaRegions.js 파일로 저장 (17개 객체 배열)
```

**시도별 hc-key → TourAPI sido_code 매핑**

Highcharts TopoJSON의 각 feature에는 `hc-key` 속성(예: `kr-so`, `kr-pu`)이 있다. 이를 백엔드 TourAPI 기준 sido_code로 변환하는 매핑 테이블을 `MyMapView.vue`에 정의했다.

```
kr-so → 1(서울),  kr-in → 2(인천),  kr-tj → 3(대전),  kr-tg → 4(대구)
kr-kj → 5(광주),  kr-pu → 6(부산),  kr-ul → 7(울산),  kr-sj → 8(세종)
kr-kg → 31(경기), kr-kw → 32(강원), kr-gb → 33(충북), kr-gn → 34(충남)
kr-cb → 35(전북), kr-2685 → 36(전남), kr-2688 → 37(경북), kr-kn → 38(경남), kr-cj → 39(제주)
```

**생성 파일**

| 파일 | 역할 |
|------|------|
| `frontend/src/assets/data/korea_sido.json` | Highcharts TopoJSON 원본 (17개 시도, WGS84 좌표) |
| `frontend/src/assets/data/koreaRegions.js` | 사전 계산된 SVG path + 레이블 좌표 배열 |
| `frontend/src/views/MyMapView.vue` | SVG 렌더링 + 방문 지역 하이라이트 |

**런타임 동작 (MyMapView.vue)**

1. `KOREA_REGIONS` 배열(koreaRegions.js)을 정적 import
2. `onMounted`에서 `GET /api/members/me/visited-regions` 호출 → 방문한 sido_code 배열 수신
3. 수신한 코드가 `visited Set`에 있으면 `region-path` 클래스에 `visited` 추가 → CSS로 보라색 표시

**현재 한계**: Highcharts TopoJSON 경계가 실제 지도와 완전히 일치하지 않아 해안선 형태에 차이 있음. 추후 SGIS(통계청 공간정보서비스) 공식 데이터로 교체 검토.

### 잠재적 버그

#### [P-1] ⏸ 보류 — N+1 쿼리 (프로필 이미지)

- **현황**: 게시글 목록 10개 = 프로필 이미지 서브쿼리 10회. 현재 데이터 규모에서 성능 이슈 없음
- **방안**: LEFT JOIN 전환. 데이터 규모가 커질 경우 진행

#### [P-2] ✅ 확인 완료 — Race Condition (좋아요 토글)

- **확인**: `schema.sql`에서 `UNIQUE KEY uq_post_like (post_id, member_id)` 확인
- **결론**: DB 레벨 UNIQUE 제약으로 중복 INSERT 자동 방지됨. 별도 코드 조치 불필요

#### [P-3] ✅ 완료 — `sort` 입력값 검증

- **결과**: `PostController`에 `@Validated` 추가, sort 파라미터에 `@Pattern(regexp = "^(latest|popular)$")` 적용
  - 허용되지 않은 값 입력 시 400 Bad Request 반환
