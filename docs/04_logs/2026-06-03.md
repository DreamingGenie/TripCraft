# 작업 일지 — 2026-06-03

## 작업자 전진

### 버그 수정 — 일정 공유 경고 팝업 미표시

`ScheduleView`의 "공유하기" 버튼으로 커뮤니티에 진입할 때 `onMounted`에서 `newPost.tripId`를 직접 할당하고 `tripShareWarning`을 `true`로 설정하지 않아 경고 팝업이 표시되지 않는 문제 수정

### 기능 개선 — 일정 공유 경고 UX 재설계

**기존 문제**

- 일정 드롭다운 선택 즉시 경고 팝업이 별도 `modal-overlay`로 렌더링되어 글쓰기 모달 위에 반투명 배경이 이중으로 쌓이는 현상
- 경고에서 확인 후 다시 등록 버튼을 눌러야 하는 2단계 구조
- 취소 시 `newPost.tripId = null` 초기화로 인해 복귀 후 재등록 시도 시 경고 없이 바로 등록되는 버그

**변경 내용**

- 경고를 별도 오버레이 대신 글쓰기 모달 내부 인라인으로 전환 → 백드랍 이중 현상 제거
- 트리거 시점 변경: 드롭다운 선택 시 → 등록 버튼 클릭 시 (`submitPost` 내부에서 `tripId` 존재 여부 확인)
- 확인 → `doSubmitPost()` 직접 호출로 즉시 등록
- 취소 → `tripShareWarning = false`만 설정, `tripId`는 유지 → 복귀 후 재시도 시 경고 재표시

**수정 파일**

- `CommunityView.vue`: 경고 렌더링 위치 변경, `submitPost` / `doSubmitPost` 분리, `cancelTripSelect` 단순화
- `community.css`: `.trip-share-warning` 인라인 스타일 추가
