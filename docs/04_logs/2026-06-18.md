## 1. DB 마이그레이션

`feature/mypage`를 머지한 뒤 반드시 아래 SQL을 실행해야 한다.  
스크립트: `docs/02_design/migration_attach_softdelete.sql`

```sql
-- 1. attach 테이블 생성 (프로필 이미지 · 게시글 첨부 메타데이터 통합 관리)
CREATE TABLE IF NOT EXISTS attach (
    id          BIGINT       NOT NULL AUTO_INCREMENT PRIMARY KEY,
    name        VARCHAR(128) NOT NULL COMMENT '서버 저장 파일명 (UUID 기반)',
    host_name   VARCHAR(128) NOT NULL COMMENT '원본 파일명',
    size        BIGINT       NOT NULL DEFAULT 0,
    mimetype    VARCHAR(128) NOT NULL,
    host_path   VARCHAR(512) NOT NULL COMMENT '서버 내 절대 경로',
    target      VARCHAR(32)  NOT NULL COMMENT 'profile | post | post_draft',
    target_id   BIGINT       NOT NULL DEFAULT 0 COMMENT '대상 레코드 ID',
    created_at  DATETIME     NOT NULL DEFAULT NOW(),
    INDEX idx_attach_target (target, target_id)
) DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_bin;

-- 2. post 소프트 딜리트 컬럼 추가
ALTER TABLE post
    ADD COLUMN deleted_at DATETIME NULL;
```

**주의**: `attach` 테이블이 없으면 프로필 이미지 API와 커뮤니티 이미지 기능 전체가 500 에러 발생.  
`deleted_at` 컬럼이 없으면 게시글 삭제 API가 실패한다.

## 5. 방문 지도 — 미완성 브리핑

### 현재 상태

라우터(`/mypage/map`)와 GNB 드롭다운 메뉴 항목이 **주석 처리**되어 있어 사용자에게 노출되지 않는다.  
`MyMapView.vue` 파일 자체는 존재하며, 기능 완성 후 주석을 해제하면 바로 활성화된다.

### 구현 방식

브라우저에서 D3 런타임 실행 시 Vue 컴포넌트 초기화 오류가 발생하는 문제가 있어, **사전 계산(Pre-compute)** 방식을 채택했다.

```
[빌드 단계 — Node.js 스크립트]
  korea_sido.json (Highcharts TopoJSON)
    → topojson-client.feature()  ← TopoJSON → GeoJSON 변환
    → d3-geo geoMercator().fitSize([500, 620])  ← 투영 설정
    → geoPath() → SVG path 문자열 + 레이블 중심 좌표
    → koreaRegions.js 저장 (17개 시도 객체 배열)

[런타임 — 브라우저]
  koreaRegions.js 정적 import (d3 불필요)
    → GET /api/members/me/visited-regions
    → 수신한 sido_code Set과 매칭 → 방문 지역 보라색 표시
```

### 주석 해제 방법

활성화할 때 수정할 위치 3곳:

```
1. frontend/src/router/index.js  — /mypage/map 라우트 주석 해제
2. frontend/src/components/AppHeader.vue  — 방문 지도 메뉴 항목 주석 해제
```

### 미해결 사항

| 항목 | 내용 |
|------|------|
| 지도 크기 | SVG가 flex 컨테이너를 채우지 못하는 CSS 레이아웃 문제. `width: 100%` 설정 후에도 개선 없음 |
| 지도 모양 | Highcharts TopoJSON이 실제 해안선과 차이 있음. SGIS(통계청) 공식 GeoJSON으로 교체 검토 필요 |
| 세종 레이블 | 세종시 면적이 작아 레이블 구분이 어려움. tooltip 방식 검토 필요 |

### 패키지 관련

`d3-geo`, `topojson-client`는 사전 계산 스크립트 전용이며 브라우저 런타임에서 import되지 않는다.  
`package.json`에서 이미 제거되었으므로 `pnpm install` 후 설치되지 않는다.  
지도 경계 데이터를 교체할 때만 이 패키지들이 다시 필요하다.

## 6. 커뮤니티 연동 사항 (마이페이지에서 영향받는 부분)

### 검색 URL 동기화

`CommunityView.vue`에서 검색어·정렬·페이지 상태를 URL 쿼리 파라미터로 관리하도록 수정했다.

- 글 상세 조회 후 뒤로가기 → 동일 검색 결과 복귀 가능
- `/community?keyword=제주&sort=popular&page=2` 형태로 URL 공유 가능

### 작성자 프로필 이미지

게시글 목록(`/community`)과 상세(`/community/:id`)에 작성자 프로필 이미지가 표시된다.  
이미지가 없으면 닉네임 첫 글자(이니셜)를 원형 배경으로 표시한다.

### 게시글 소프트 딜리트

`post.deleted_at`이 non-NULL이면 '삭제된 게시글'로 표시되고 title은 null로 반환된다.  
북마크·좋아요 목록에서 삭제된 글도 행 자체는 유지되며, `deleted: true` 플래그로 UI를 분기한다.
