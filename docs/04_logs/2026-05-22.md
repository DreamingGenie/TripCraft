### 2. DB 스키마 DDL 설계 (`docs/02_design/schema.sql`)

의논을 통해 결정한 주요 설계 사항:

- **테이블 12개**: `member`, `member_token`, `attraction`, `favorite`, `trip`, `trip_candidate`, `trip_block`, `transit_cache`, `system_config`, `post`, `post_like`, `notice`
- **회원**: 하드 딜리트, 탈퇴 시 앱 레이어에서 순서대로 삭제 (`trip_block` → `trip_candidate` → `trip` → `member`)
- **게시글**: 회원 탈퇴 시 보존, `post.member_id SET NULL` (탈퇴한 사용자로 표시)
- **공유 중인 일정 삭제 불가**: `post.trip_id` 참조 존재 시 애플리케이션 레이어에서 차단
- **transit_cache 키**: `(from_attraction_id, to_attraction_id, departure_hour, transport_type)` — 레벨 무관 공유
- **캐시 레벨 1~5**: 관리자 `system_config`로 설정, 대표 시간대별 슬롯 수 1/2/3/5/24개
- **이동수단**: 대중교통(전체·지하철·버스) + 자동차 (`transport_type` 0~3)
- **trip_block.display_order**: UNIQUE 제약 없음 (Phase 2 실시간 편집 고려)
- **날짜 범위 검증**: `trip_block` INSERT/UPDATE 시 TRIGGER로 여행 기간 체크

### 6. JWT 보안 설정 및 Auth API 구현

| 파일 | 역할 |
|------|------|
| `JwtTokenProvider` | Access(30분) / Refresh(7일) 토큰 발급·검증 |
| `JwtAuthenticationFilter` | Authorization Bearer 헤더 파싱, SecurityContext 설정 |
| `SecurityConfig` | Stateless, 공개·보호·관리자 엔드포인트 분리, CORS 설정 |
| `AuthEntryPoint` | 401 → 공통 응답 형식 |
| `AccessDeniedHandlerImpl` | 403 → 공통 응답 형식 |
| `AuthController` | signup / login / refresh / logout 엔드포인트 |
| `AuthServiceImpl` | 비밀번호 BCrypt 검증, 토큰 발급·갱신·무효화 |

**Swagger** (`springdoc-openapi`) 추가 — `http://localhost:8080/swagger-ui/index.html`

**트러블슈팅**:
- `member_token` 테이블 컬럼명 `refresh_token` ↔ 도메인 필드 `token` 불일치 → XML resultMap 수정으로 해결
- CORS preflight(OPTIONS) 인증 차단 → `CorsConfigurationSource` 빈 등록 + OPTIONS 허용으로 해결
- Swagger 경로 인증 차단 → SecurityConfig permitAll 경로 추가로 해결

### 7. 프론트엔드 프로젝트 구조 초기화 (`frontend/`)

단일 레포지토리 내 `backend/` · `frontend/` · `docs/` 폴더 분리 구조 확립.  
현재는 Vanilla HTML 기반이며, 추후 Vue 3 + Vite 마이그레이션 예정.

**폴더 구조**
```
frontend/
├── assets/
│   ├── css/
│   │   ├── variables.css   # CSS 디자인 토큰
│   │   ├── common.css      # GNB, 버튼, 모달, 토스트 공통
│   │   ├── explore.css
│   │   ├── schedule.css
│   │   ├── community.css
│   │   └── auth.css
│   └── js/
│       ├── common.js       # 토스트, 일정 생성 모달, GNB 인증 상태
│       ├── auth.js         # 로그인/회원가입 API 연동
│       ├── explore.js
│       ├── schedule.js
│       └── community.js
└── pages/
    ├── explore.html
    ├── schedule.html
    ├── community.html
    └── auth.html
```

DB 테이블 구조 기반으로 전면 재구현:

- **목록 뷰**: 게시글 카드 (제목/작성자/날짜/요약/태그/통계), 정렬 버튼, 페이지네이션, 사이드바(공지 5건·이번 주 인기 5건)
- **상세 뷰**: 본문, 좋아요 버튼, 수정/삭제(조건부), 댓글 목록(대댓글·댓글 좋아요·작성자 뱃지)
- **글쓰기 모달**: 제목, 내용, 여행 일정 연결(선택)

### 10. 로그인/회원가입 API 연동 (`auth.js`)

- 로그인: POST `/api/auth/login` → Access Token 수신 (저장 방식 미결, 현재 localStorage 임시 사용)
- 회원가입: POST `/api/auth/signup` → 토큰 미발급, 성공 시 로그인 화면으로 이동
- 에러 처리: 409(이메일 중복) / 401(인증 실패) / 400(입력값 오류) 별도 toast
- GNB: `initGnb()` 로 로그인 상태에 따라 로그인/회원가입 버튼 ↔ 닉네임+로그아웃 동적 렌더링
