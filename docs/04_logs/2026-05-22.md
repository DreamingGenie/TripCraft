### 2. DB 스키마 DDL 설계 (`docs/02_design/schema.sql`)

의논을 통해 결정한 주요 설계 사항:

- **테이블 12개**: `member`, `member_token`, `attraction`, `favorite`, `trip`, `trip_candidate`, `trip_block`, `transit_cache`, `system_config`, `post`, `post_like`, `notice`
- **회원**: 하드 딜리트, 탈퇴 시 앱 레이어에서 순서대로 삭제 (`trip_block` → `trip_candidate` → `trip` → `member`)
- **게시글**: 회원 탈퇴 시 보존, `post.member_id SET NULL` (탈퇴한 사용자로 표시)
- **공유 중인 일정 삭제 불가**: `post.trip_id` 참조 존재 시 애플리케이션 레이어에서 차단
- **transit_cache 키**: `(from_attraction_id, to_attraction_id, departure_hour, transport_type)` — 레벨 무관 공유
- **캐시 레벨 1~5**: 관리자 `system_config`로 설정, 대표 시간대별 슬롯 수 1/2/3/5/24개
- **이동수단**: 대중교통(전체·지하철·버스) + 자동차 (`transport_type` 0~3)
- **trip_block.display_order**: UNIQUE 제약 없음 (Phase 2 실시간 편집 고려)
- **날짜 범위 검증**: `trip_block` INSERT/UPDATE 시 TRIGGER로 여행 기간 체크

### 6. 프론트엔드 프로젝트 구조 초기화 (`frontend/`)

단일 레포지토리 내 `backend/` · `frontend/` · `docs/` 폴더 분리 구조 확립.  
현재는 Vanilla HTML 기반이며, 추후 Vue 3 + Vite 마이그레이션 예정.

**폴더 구조**
```
frontend/
├── assets/
│   ├── css/
│   │   ├── variables.css   # CSS 디자인 토큰 (컬러, 간격, 폰트 변수)
│   │   ├── common.css      # GNB, 버튼, 칩, 모달, 토스트 공통
│   │   ├── explore.css     # 관광지 탐색 페이지 전용
│   │   ├── schedule.css    # 내 일정 페이지 전용
│   │   ├── community.css   # 커뮤니티 페이지 전용
│   │   └── auth.css        # 로그인/회원가입 페이지 전용
│   └── js/
│       ├── common.js       # 토스트, 일정 생성 모달 공통 유틸
│       ├── explore.js      # 필터 토글, 일정 선택, 장소 추가 로직
│       └── schedule.js     # 시간표 그리드 생성, 드래그&드롭
└── pages/
    ├── explore.html        # 관광지 탐색
    ├── schedule.html       # 내 일정
    ├── community.html      # 커뮤니티
    └── auth.html           # 로그인/회원가입
```

### 7. 화면 초안 수정 — 관광지 탐색 페이지 (`explore.html`)

| 변경 전 | 변경 후 |
|--------|--------|
| 왼쪽 사이드바: 지역·카테고리·즐겨찾기 필터 | 왼쪽 사이드바: 내 일정 목록 |
| 필터가 항상 노출 | 검색창 하단 버튼 클릭 시 필터 패널 토글 |
| 새 일정 만들기 버튼 위치: 탐색 페이지 | 새 일정 만들기 버튼 위치: 내 일정 페이지로 이동 |

**내 일정 목록 카드 구성 요소**
- 일정 이름, 여행 인원, 시작일 ~ 종료일
- 지역별 추가된 장소 칩 목록
- 선택 버튼 (어떤 일정에 장소를 추가할지 지정)

### 8. 화면 초안 수정 — 내 일정 페이지 (`schedule.html`)

| 변경 전 | 변경 후 |
|--------|--------|
| Day 1 / Day 2 / Day 3 탭 전환 방식 | 날짜별 컬럼 × 0~24시 행 시간표 그리드 |
| 날짜별 타임라인 세로 나열 | 날짜가 좌우로 배치, 시간이 위아래로 배치 |
| 드롭존이 날짜별로 분리 | 모든 날짜를 한 화면에서 비교하며 편집 가능 |

**시간표 그리드 상세**
- 1시간 = 60px, 30분 단위 스냅으로 드래그&드롭
- 시간 축(좌): sticky left, 날짜 헤더(상): sticky top → 스크롤 시 항상 노출
- 초기 스크롤 위치: 오전 8시 (480px)
- 이동 시간 블록: 빗금 패턴으로 이벤트 사이에 표시
- 새 일정 만들기 버튼 및 모달을 툴바에 배치
