### 2. DB 스키마 DDL 설계 (`docs/02_design/schema.sql`)

의논을 통해 결정한 주요 설계 사항:

- **테이블 12개**: `member`, `member_token`, `attraction`, `favorite`, `trip`, `trip_candidate`, `trip_block`, `transit_cache`, `system_config`, `post`, `post_like`, `notice`
- **회원**: 하드 딜리트, 탈퇴 시 앱 레이어에서 순서대로 삭제 (`trip_block` → `trip_candidate` → `trip` → `member`)
- **게시글**: 회원 탈퇴 시 보존, `post.member_id SET NULL` (탈퇴한 사용자로 표시)
- **공유 중인 일정 삭제 불가**: `post.trip_id` 참조 존재 시 애플리케이션 레이어에서 차단
- **transit_cache 키**: `(from_attraction_id, to_attraction_id, departure_hour, transport_type)` — 레벨 무관 공유
- **캐시 레벨 1~5**: 관리자 `system_config`로 설정, 대표 시간대별 슬롯 수 1/2/3/5/24개
- **이동수단**: 대중교통(전체·지하철·버스) + 자동차 (`transport_type` 0~3)
- **trip_block.display_order**: UNIQUE 제약 없음 (Phase 2 실시간 편집 고려)
- **날짜 범위 검증**: `trip_block` INSERT/UPDATE 시 TRIGGER로 여행 기간 체크

### 5. JWT 보안 설정 구현

| 파일 | 역할 |
|------|------|
| `JwtTokenProvider` | Access(30분) / Refresh(7일) 토큰 발급·검증 |
| `JwtAuthenticationFilter` | Authorization Bearer 헤더 파싱, SecurityContext 설정 |
| `SecurityConfig` | Stateless, 공개·보호·관리자 엔드포인트 분리 |
| `AuthEntryPoint` | 401 → 공통 응답 형식 |
| `AccessDeniedHandlerImpl` | 403 → 공통 응답 형식 |
| `ApiResponse<T>` | 공통 응답 래퍼 |
| `Member` / `MemberMapper` | 회원 도메인, MyBatis XML Mapper |
