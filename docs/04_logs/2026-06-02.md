### 버그 수정 — `-parameters` 컴파일 플래그 누락

#### 현상

백엔드 구동 후 일정·게시글 로드 실패.
`IllegalArgumentException: Name for argument of type [Long] not specified` 에러 반복.

#### 원인

VSCode Spring Boot Dashboard는 Gradle이 아닌 VSCode 내장 Java 컴파일러(ECJ)로 `bin/main`을 컴파일한다.
ECJ는 기본적으로 `-parameters` 플래그 없이 컴파일하므로 Spring이 `@PathVariable`, `@RequestParam` 파라미터 이름을 reflection으로 읽지 못한다.

#### 수정

모든 컨트롤러의 `@PathVariable`, `@RequestParam`에 명시적 이름 추가.

| 파일 | 수정 항목 |
|------|----------|
| `PostController` | `@PathVariable Long id` → `@PathVariable("id") Long id` (3곳) |
| `CommentController` | `@PathVariable Long postId/commentId` → 명시적 name (4곳) |
| `TripController` | `@PathVariable Long id/blockId/candidateId/tripId` → 명시적 name (9곳) |
| `AttractionController` | `@PathVariable Long id` → `@PathVariable("id") Long id` (1곳) |

**버그 수정 — 시간 표시 9시간 오차**

Spring `LocalDateTime`이 타임존 없이 직렬화되면 브라우저가 KST로 해석해 9시간 차이 발생.
`formatDate()`에서 타임존 정보 없는 문자열에 `'Z'`(UTC)를 붙여 파싱하도록 수정.

## 송정기 작업

### 1. TMap 통행료 필드명 수정 (`tollFare` → `totalFare`)

#### 현상

서울→강릉 경로에서 통행료가 표시되지 않음.

#### 원인

`TMapClient.java`에서 `props.path("tollFare")`를 읽었으나, TMap prediction API 응답에서 실제 필드명은 `totalFare`.

#### 해결

`props.path("totalFare").asInt(0)`으로 수정. 디버그용 `log.info("TMap Point props: {}")` 제거.

### 2. 자동차 경로 요약(roadSummary) 로직 개선

#### 변경 전

전체 세그먼트에서 중복 제거 후 순서대로 최대 4개 선택.

#### 변경 후

- **선택 기준**: 고속도로(`roadType == 1`)를 `distanceM` 내림차순으로 우선 선택, 4개 미만이면 비고속도로로 채움
- **표시 순서**: 선택된 세그먼트를 원래 경로 순서대로 표시

### 4. 수동 저장된 `transitMode` 재계산 시 원복 버그 수정

#### 현상

DRIVING·WALKING으로 저장한 이동수단이 블록 드래그/리사이즈 후 PUBLIC_TRANSIT으로 되돌아옴.

#### 원인

`recalculateTransitForDate()`가 드래그/리사이즈 시 호출되면서 모든 블록의 `transitMode`를 trip 기본값(PUBLIC_TRANSIT)으로 덮어씀.

#### 해결

`block.getTransitMode() != null`인 경우 해당 모드를 유지(`effectiveMode`로 사용)하고 소요시간만 재계산.

```java
String effectiveMode = block.getTransitMode() != null ? block.getTransitMode() : transitMode;
Optional<TransitResponse> transit = transitService.getTransitTime(fromAttrId, toAttrId, 9, effectiveMode);
block.setTransitDurationMinutes(transit.map(TransitResponse::getDurationMinutes).orElse(null));
if (block.getTransitMode() == null) {
    block.setTransitMode(transit.map(TransitResponse::getTransportMode).orElse(null));
}
```

### 5. 대중교통 경로 좌표(routeCoords) 지도 표시 지원

#### 배경

PUBLIC_TRANSIT은 `routeCoords`가 저장되지 않아 지도에서 직선 점선으로만 표시됨.

#### 해결

ODsay `subPath`의 `startX/Y`, `endX/Y`를 추출해 `[[lng, lat], ...]` 형태로 저장.
별도 API 호출 없이 기존 응답 데이터 활용.

> **주의**: 기존 캐시(`transit_cache`)에는 `routeCoords`가 null로 저장됨. 새로 조회하는 경로부터 반영.
