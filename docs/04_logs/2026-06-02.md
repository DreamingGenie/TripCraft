### 버그 수정 — `-parameters` 컴파일 플래그 누락

---

#### 현상

백엔드 구동 후 일정·게시글 로드 실패.
`IllegalArgumentException: Name for argument of type [Long] not specified` 에러 반복.

---

#### 원인

VSCode Spring Boot Dashboard는 Gradle이 아닌 VSCode 내장 Java 컴파일러(ECJ)로 `bin/main`을 컴파일한다.
ECJ는 기본적으로 `-parameters` 플래그 없이 컴파일하므로 Spring이 `@PathVariable`, `@RequestParam` 파라미터 이름을 reflection으로 읽지 못한다.
`build.gradle.kts`에 이미 `tasks.withType<JavaCompile> { options.compilerArgs.add("-parameters") }`가 있었으나 ECJ에는 적용되지 않았다.

---

#### 수정

코드에서 직접 파라미터 이름을 명시하는 방식으로 해결. 모든 컨트롤러의 `@PathVariable`, `@RequestParam`에 명시적 이름 추가.

| 파일 | 수정 항목 |
|------|----------|
| `PostController` | `@PathVariable Long id` → `@PathVariable("id") Long id` (3곳) |
| `CommentController` | `@PathVariable Long postId/commentId` → 명시적 name (4곳) |
| `TripController` | `@PathVariable Long id/blockId/candidateId/tripId` → 명시적 name (9곳) |
| `AttractionController` | `@PathVariable Long id` → `@PathVariable("id") Long id` (1곳) |

#### 버그 수정 — 시간 표시 9시간 오차

**현상**: 게시글 작성 직후 "9시간 전"으로 표시됨.

**원인**: Spring `LocalDateTime`이 타임존 없이 직렬화(`"2026-06-02T05:01:23"`)되면 브라우저의 `new Date()`가 이를 로컬 시간(KST)으로 해석. 실제 UTC 값과 9시간 차이 발생.

**수정**: `formatDate()` 함수에서 타임존 정보 없는 문자열에 `'Z'`(UTC)를 붙여 파싱.

```js
const d = new Date(dt.includes('Z') || dt.includes('+') ? dt : dt + 'Z')
```
