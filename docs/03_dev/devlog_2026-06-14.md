## 작업자 전진

### DB 마이그레이션 (attach 테이블 신설 + soft delete)

- `attach` 테이블 신설: 파일 메타데이터 통합 관리 (`target`, `target_id`로 대상 구분)
- `post.deleted_at DATETIME NULL` 컬럼 추가 (소프트 딜리트)
- `post_bookmark` 테이블 신설: `post_id` FK에 ON DELETE CASCADE 미적용 (소프트 딜리트 연동)
- 관련 파일: `docs/02_design/migration_attach_softdelete.sql`, `schema.sql` v0.5 갱신

### attach 백엔드 연동

- `Attach` 도메인, `AttachMapper`, `AttachMapper.xml` 신설
- `ImageController`: 이미지 업로드 시 `target='post_draft'` 레코드 자동 생성
- `PostServiceImpl.createPost()`: 글 저장 완료 후 `post_draft → post` bulk UPDATE
- `PostServiceImpl.deletePost()`: 물리 삭제 → 소프트 딜리트(`softDeleteById`)로 전환
- `PostMapper.xml`: 조회 쿼리 전반에 `deleted_at IS NULL` 조건 추가

### 북마크 기능

- `POST /api/posts/{id}/bookmark`: 북마크 토글
- `GET /api/bookmarks/me`: 내 북마크 목록 (페이지네이션)
- 삭제된 글은 `CASE WHEN` 처리로 "삭제된 글입니다" 표시
- `PostDetail.bookmarked` 필드 추가 (글 상세 조회 시 북마크 여부 반환)
- 프론트: `MyBookmarksView.vue`, `/mypage/bookmarks` 라우트, 글 상세 북마크 버튼

### 좋아요한 글 목록

- `GET /api/likes/me`: 내가 좋아요한 글 목록 (페이지네이션)
- `PostLikeMapper`에 `findByMemberId` / `countByMemberId` 쿼리 추가
- 소프트 딜리트 게시글 "삭제된 글입니다" 처리 포함
- 프론트: `MyLikesView.vue`, `/mypage/likes` 라우트

### 내가 쓴 글 목록

- `GET /api/posts/me`: 내 게시글 목록 (페이지네이션)
- `PostService.getMyPosts()` 구현, `PostMapper.findByMemberId` 쿼리 활용
- 목록에서 바로 삭제 가능 (소프트 딜리트 연동)
- 프론트: `MyPostsView.vue`, `/mypage/posts` 라우트

### 내 정보 수정

- `PATCH /api/members/me/nickname`: 닉네임 변경
- `PATCH /api/members/me/password`: 현재 비밀번호 검증 후 신규 비밀번호 변경 (BCrypt 인코딩)
- `MemberMapper`에 `updateNickname` / `updatePassword` 쿼리 추가
- `UpdateNicknameRequest`, `UpdatePasswordRequest` DTO 신설
- 프론트: `MyProfileView.vue`, `/mypage/profile` 라우트, 닉네임 변경 시 Pinia store 즉시 반영

### 방문 지도

- `GET /api/members/me/visited-regions`: trip → trip_candidate → attraction 조인으로 방문 sido_code 집계
- `TripMapper.findVisitedSidoCodes()` 쿼리 추가
- 프론트: `MyMapView.vue`, `/mypage/map` 라우트
- 17개 시도를 격자형 SVG 타일로 표현, 방문 지역 하이라이트

### GNB 드롭다운

- `AppHeader.vue`: 닉네임 버튼 클릭 시 마이페이지 서브메뉴 표시
- 메뉴 항목: 내 정보 수정 / 방문 지도 / 내가 쓴 글 / 북마크 / 좋아요한 글
- 외부 클릭 감지로 자동 닫힘 (`capture` 단계 이벤트 리스너)
