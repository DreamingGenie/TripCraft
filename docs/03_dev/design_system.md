# TripCraft 디자인 시스템 — "Warm Editorial Workspace"

> 전 화면을 하나의 언어로 묶기 위한 기준 문서.
> 화면은 **작업 공간(workspace)** 과 **콘텐츠(content)** 두 밀도로 나뉘지만,
> **같은 토큰·컴포넌트·아이콘·색**에서 나온다.

---

## 1. 화면 밀도 분류

| 밀도 | 화면 | 특징 |
|------|------|------|
| **workspace** (dense) | 관광지 탐색, 내 일정 | 지도+패널+드래그, 정보 밀집, 오래 머무름 |
| **content** (airy) | 여행일지, 커뮤니티, 마이페이지 | 읽고 둘러보는 화면, 여백 넉넉 |

밀도는 **여백·폰트 크기**로만 조절하고, 색·아이콘·컴포넌트 모양은 공유한다.

---

## 2. 디자인 원칙 (Warm Editorial)

1. **위계는 타이포로, 색은 절제** — 영역마다 컬러 박스를 쌓지 말고 크기·굵기·들여쓰기로 단계를 만든다. 퍼플은 "선택/강조"에만.
2. **칸막이(테두리)보다 면(그림자)** — 0.5px 헤어라인 남발 금지. `--shadow-sm/md`로 카드가 떠 보이게.
3. **따뜻한 종이 베이스** — 배경 `#F5F4F0`, surface 순백. 차가운 회색 지양.
4. **아이콘은 한 세트(라인 SVG)** — 이모지 금지(분류 색 점/뱃지는 예외적으로 허용하지 않음, 색 토큰으로 표현).
5. **분류 색은 전 화면 공통** — 관광 콘텐츠타입 색이 지도 핀·카드·일정 블록·달력 띠에서 동일해야 한다.

---

## 3. 토큰

### 3.1 타이포 스케일 (신규 — `--text-*`)
새 화면은 이 스케일만 쓴다. 기존 `--font-size-*`(10~14px)는 화면 마이그레이션이 끝나면 폐기.

| 토큰 | px | 용도 |
|------|----|------|
| `--text-2xs` | 11 | 마이크로 라벨, 카운트 |
| `--text-xs`  | 12 | 보조 메타 |
| `--text-sm`  | 13 | 본문 보조, 테이블 |
| `--text-base`| 14 | 본문 기본, 입력 |
| `--text-md`  | 15 | 카드 제목 |
| `--text-lg`  | 16 | 소제목 |
| `--text-xl`  | 18 | 패널 제목 |
| `--text-2xl` | 22 | 상세 제목 |
| `--text-3xl` | 28 | 페이지 히어로 |

### 3.2 분류 색 (콘텐츠타입 = 고정 색)
카드 배경은 `tint`, 핀·뱃지·일정블록은 `ink`.

| 콘텐츠타입 | id | 계열 | tint | ink |
|-----------|----|------|------|-----|
| 관광지 | 12 | indigo | `--cat-sights-tint` | `--cat-sights-ink` |
| 문화시설 | 14 | teal | `--cat-culture-tint` | `--cat-culture-ink` |
| 레포츠 | 28 | green | `--cat-leisure-tint` | `--cat-leisure-ink` |
| 숙박 | 32 | blue | `--cat-stay-tint` | `--cat-stay-ink` |
| 쇼핑 | 38 | amber | `--cat-shop-tint` | `--cat-shop-ink` |
| 음식점 | 39 | pink | `--cat-food-tint` | `--cat-food-ink` |

> 현재 `ExploreView`의 JS `COLORS` 맵이 이 tint 값과 일치해야 한다. Schedule/Calendar도 동일 색을 쓴다.

### 3.3 그 외
- 간격: `--space-1..6` (4/8/12/16/24/32)
- 라운드: 컨트롤 `--radius-md`(6), 카드 `--radius-xl`(12), 패널 `--radius-xl`, 칩/뱃지 `--radius-full`
- 그림자: `--shadow-sm/md/lg`

---

## 4. 공통 컴포넌트 (점진 도입)

| 컴포넌트 | 역할 | 대체 대상 |
|---------|------|----------|
| `Icon` | 라인 SVG 단일 세트 | 이모지(🔍🗺📢▶✕…) |
| `Chip` | 필터/태그 | `.chip` 산재 정의 |
| `Button` | primary/ghost/danger | 뷰별 버튼 |
| `Panel` | 플로팅 카드 크롬(그림자·라운드·헤더) | attr-list·detail·tray 중복 |
| `SectionHeader` | 접기형 섹션 제목 | 25종 `*-header` |
| `EmptyState` | 빈 상태 | `*-empty` 산재 |

---

## 5. 마이그레이션 순서

1. 토큰·스펙 확정(이 문서) ✅
2. **관광지 탐색** 시범 적용 ← 현재
3. 내 일정 → 여행일지 → 커뮤니티 → 마이페이지
4. 화면 안정화 후 `--font-size-*` 레거시 토큰·중복 `*-header` 제거, stylelint로 raw hex/px 차단
