## 2. 기획 배경

### 2-1. 문제 인식

국내 여행을 계획할 때 사용자는 **최소 4~5개의 서로 다른 서비스**를 동시에 열어두고 작업해야 한다.

가고 싶은 관광지를 포털 지도로 검색하고, 운영 시간과 예약 가능 여부는 개별 사이트에서 확인하며, 장소 사이의 이동 수단과 소요 시간은 대중교통 앱으로 따로 계산하고, 숙소는 예약 플랫폼에서 비교한다. 일행이 있다면 이 모든 과정을 메신저와 노션, 공유 메모 등 여러 채널을 오가며 맞춰가야 한다.

결국 "여행 계획을 세우는 일" 자체가 피로하고 복잡한 작업이 된다. 세심하게 짠 일정도 막상 현장에서 이동 시간이 예상과 달라지면 전체 흐름이 무너지는 경우가 빈번하다.

### 2-2. 핵심 불편함

- **정보의 분산**: 관광지·이동·숙소 정보가 모두 다른 서비스에 흩어져 있어 통합된 뷰를 얻을 수 없다.
- **수동 이동 시간 계산**: 장소가 바뀔 때마다 이동 시간을 직접 검색하고 일정에 수기로 반영해야 한다.
- **일정의 경직성**: 예약 장소의 방문 시간이 고정되어 있어도, 전후 일정과의 연계를 자동으로 재배치해 주는 도구가 없다.
- **협업의 불편함**: 노션·스프레드시트 등 범용 도구는 여행 일정 특유의 시간·장소 구조를 표현하기 어렵고, 일행과 실시간으로 논의하며 수정하기에 적합하지 않다.

### 2-3. 해결 방향

TripCraft는 이 네 가지 불편함을 하나의 플랫폼 안에서 해소하는 것을 목표로 한다.

사용자는 먼저 **여행 일정(날짜 범위)을 생성**한 뒤, 관광지 탐색에서 가고 싶은 장소를 골라 일정의 **후보군**으로 등록한다. 후보군은 장소가 위치한 **도시 단위로 자동 분류**되어 도시별 묶음으로 표시된다. 즐겨찾기로 저장해 둔 장소는 해당 도시가 후보군에 처음 등록되는 순간 자동으로 함께 추가된다. 이후 **내 일정** 탭에서 도시별 후보군 사이드바를 보며 원하는 장소를 특정 날짜·시간 슬롯에 드래그 앤 드롭으로 확정한다. 장소를 배치하면 ODsay API 기반 이동 시간이 자동으로 계산·삽입된다.

## 3. 서비스 목표

### 3-1. 핵심 가치 (Core Value)

| 가치 | 설명 |
|------|------|
| **통합(Integration)** | 관광지 탐색 → 후보군 등록 → 일정 확정 → 공유까지 하나의 플로우로 |
| **자동화(Automation)** | 장소 추가 시 도시 분류·이동 시간·즐겨찾기 연동이 자동 처리됨 |
| **유연성(Flexibility)** | 후보군을 먼저 모아두고 날짜·시간은 나중에 결정하는 2단계 계획 방식 |
| **협업(Collaboration)** | 여행 일정 특화 UI로 일행과 실시간 공동 편집 (Phase 2) |

### 3-2. 목표 사용자

- 국내 여행지를 직접 조사하고 일정을 짜는 **자유여행 선호자**
- 2인 이상이 함께 여행 계획을 세우는 **소그룹 여행자**
- 촘촘한 일정보다는 유연한 플로우를 선호하지만, **기본 동선은 미리 잡아두고 싶은** 여행자

## 4. 핵심 사용자 플로우

```
[1단계 — 일정 생성]
  → 여행 제목·날짜 범위·인원 입력 후 일정 생성

[2단계 — 후보군 등록] (관광지 탐색 탭)
  → 지역·카테고리 필터로 관광지 탐색
  → 장소 카드에서 "일정에 추가" 클릭
  → 해당 장소가 일정의 후보군에 등록됨
  → 장소가 위치한 도시(시·군 단위)가 후보군에 처음 추가되는 경우,
     즐겨찾기에 등록된 동일 도시 장소들이 후보군에 자동 추가됨
  → 후보군은 도시별로 자동 묶임 (예: 서울 · 경주 · 제주)

[3단계 — 일정 확정] (내 일정 탭)
  → 날짜별 빈 시간표 + 도시별 후보군 사이드바 노출
  → 후보군에서 장소를 특정 날짜·시간 슬롯으로 드래그 앤 드롭
  → 배치 완료 시 ODsay API로 이동 시간 자동 계산 및 삽입

[4단계 — 공유] (커뮤니티 탭)
  → 완성된 일정을 게시판에 공유
```

## 5. MVP 기능 정의

> **Phase 1 (MVP)** 과 **Phase 2 (고도화)** 로 구분합니다.

### Phase 1 — MVP (필수 구현)

| ID | 기능 영역 | 기능명 | 설명 |
|----|----------|--------|------|
| F01 | 관광지 | 관광 DB 구축 | 한국관광공사 국문 관광정보 API로 관광지 데이터 수집 및 저장 |
| F02 | 관광지 | 지역별 조회 | 시도·시군구 단위 필터링으로 관광지 목록 제공 |
| F03 | 관광지 | 카테고리별 조회 | 관광지·숙박·음식점·문화시설 등 콘텐츠 타입별 조회 |
| F04 | 관광지 | 즐겨찾기 | 장소별 즐겨찾기 저장·해제. 일정에 해당 도시가 처음 추가될 때 동일 도시 즐겨찾기 장소가 후보군에 자동 추가됨 |
| F05 | 일정 | 일정 생성 | 여행 제목·출발일·귀환일·인원을 입력해 일정 생성 |
| F06 | 일정 | 후보군 등록 | 탐색 화면에서 장소를 일정의 후보군으로 추가. 후보군은 도시별로 자동 분류 |
| F07 | 일정 | 일정 확정 (드래그 앤 드롭) | 내 일정 탭에서 후보군 장소를 날짜·시간 슬롯에 드래그 앤 드롭으로 배치 |
| F08 | 일정 | 이동 시간 자동 계산 | 장소 배치 시 ODsay API로 대중교통·도보 이동 시간 자동 삽입 |
| F09 | 일정 | 일정 저장·불러오기 | 후보군 및 확정 일정을 저장하고 재편집 |
| F10 | 커뮤니티 | 여행 일정 공유게시판 | 완성된 일정을 게시판에 공유, 다른 사용자의 일정 열람 |
| F11 | 커뮤니티 | 공지사항 | 관리자 공지 CRUD |
| F12 | 회원 | 회원 관리 | 회원가입·로그인·로그아웃·정보수정·탈퇴 |

> **기존 ID 변경 안내**: v0.1의 F05(블록형 시간표)·F06(이동 시간)·F07(저장)이 v0.2에서 F05~F09로 세분화됨.

### Phase 2 — 고도화 (MVP 안정화 후 추가)
