# 작업 일지 — 2026-06-04

## 작업자 전진

### feat(calendar): 내 여행 일지 캘린더 페이지 구현

#### 배경

여행 일정이 누적될수록 "언제 어디를 다녀왔는지" 전체 흐름을 한 눈에 파악하기 어려웠음.
기존 `/schedule` 페이지는 단일 여행 편집에 최적화된 구조라 이력 탐색 용도에 부적합.
월별 캘린더 뷰를 별도 페이지로 분리해 여행 이력 조회 기능을 추가.

#### 설계 결정

**백엔드 API 추가 없음**: 이미 존재하는 두 엔드포인트로 필요한 데이터를 모두 충족.

| 사용 API | 목적 |
|----------|------|
| `GET /api/trips` | 전체 여행 목록 (startDate, endDate 포함) |
| `GET /api/trips/{id}/blocks-summary` | 날짜별 방문지 목록 |

`blocks-summary`는 날짜 클릭 시에만 요청하며 결과를 컴포넌트 내 `blockCache`에 캐싱. 동일 여행을 다른 날짜에서 재클릭해도 API 재호출 없음.

#### 구현 내용

**라우팅 및 진입점**

- `/calendar` 라우트 추가 (`requiresAuth: true`)
- GNB에 "내 여행 일지" 링크 추가 (내 일정 ~ 커뮤니티 사이)

**캘린더 그리드**

- 이전 달 / 다음 달의 날짜도 회색으로 표시해 주(週) 행이 항상 7칸 유지
- 오늘 날짜: 보라색 원형 배경 표시
- 일요일·토요일: 빨강·파랑으로 구분

**월 네비게이션**

- 좌우 화살표로 월 이동
- 월 타이틀 클릭 → 연도(±1 버튼) + 12개 월 버튼 피커 드롭다운

**여행 띠(strip) 렌더링**

연속된 일정은 첫 번째 이미지의 방탈출 캘린더처럼 끊김 없는 띠로 표시.

- 시작일: 왼쪽 반원 모서리 + 여행 제목 텍스트
- 중간 날: 직사각형 연장
- 마지막 날: 오른쪽 반원 모서리
- 월 경계에서 주(週) 행이 끊기면 다음 행에서 직사각형으로 이어짐

겹치는 여행은 레인(행)을 분리해 표시. 레인 배정 알고리즘:

```
각 여행을 순서대로 순회
→ 현재 레인에 이미 배정된 스트립과 열(column) 범위가 겹치면 다음 레인으로
→ 빈 레인이 없으면 새 레인 추가
```

여행 색상은 `trips` 배열 인덱스 % 6으로 보라·청록·분홍·파랑·앰버·초록 순환.
색상 배정은 로드 순서 기준이므로, 여행을 삭제·재생성하면 색이 바뀔 수 있음.

**날짜 클릭 → 상세 패널**

- 해당 날짜를 포함한 모든 여행을 우측 사이드 패널에 표시
- 각 여행의 확정 방문지 목록: 시작 시간 · 장소명 · 체류 시간
- 패널은 슬라이드 트랜지션으로 열림/닫힘
- 모바일(768px 이하)에서는 패널이 하단으로 이동

#### 사용 흐름

```
GNB "내 여행 일지" 클릭
  → 오늘 날짜 기준으로 캘린더 페이지 오픈
  → 여행 기간이 띠로 표시됨
  → 날짜 클릭 → 우측 패널에 방문지 목록 표시
  → 화살표 or 월 타이틀 클릭으로 다른 달 이동
```

#### 알려진 제약

- `blocks-summary`는 게시글에 공유된 일정만 비인증 허용. 캘린더는 인증 후 본인 일정을 조회하는 구조라 현재 보안 설정과 충돌 없음.
- 블록 캐시는 페이지 세션 내에서만 유효. 다른 페이지 이동 후 복귀하면 재요청.
