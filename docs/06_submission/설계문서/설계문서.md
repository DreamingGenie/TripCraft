/* 파일 경계 페이지 나눔 */

@media print {
  .page-break { page-break-after: always; }

  /* 다이어그램·표·코드·이미지 자체는 중간에서 쪼개지지 않게 */
  .mermaid, svg, img, pre, table, blockquote {
    break-inside: avoid;
    page-break-inside: avoid;
  }

  /* 제목은 바로 뒤 내용과 붙어 함께 넘어감 */
  h1, h2, h3, h4, h5, h6 {
    break-after: avoid;
    page-break-after: avoid;
  }

  /*
    핵심: 그림(머메이드/이미지/표/코드)이 다음 페이지로 밀릴 때
    그 "바로 앞의 제목·단락·인용"도 함께 끌고 가서, 윗페이지에 제목만 덩그러니
    남고 큰 여백이 생기는 문제를 막는다. (제목→인트로 단락→그림이 한 덩어리로 이동)
    Chrome(Puppeteer) 내보내기 기준 :has() 지원.
  */
  *:has(+ .mermaid),
  *:has(+ svg),
  *:has(+ p > img),
  *:has(+ p > svg),
  *:has(+ pre),
  *:has(+ table),
  *:has(+ blockquote) {
    break-after: avoid;
    page-break-after: avoid;
  }
}

/*
  mermaid·SVG·이미지를 페이지 "폭"뿐 아니라 "높이"까지 한 페이지 안에 강제로 맞춘다.
  - MPE가 svg 에 inline max-width 를 박아두므로 !important 로 덮어써야 함.
  - max-height 를 페이지 높이(vh) 기준으로 제한 → 한 페이지를 넘던 큰 다이어그램은
    비율을 유지한 채 자동 축소되어 한 페이지에 들어온다. (제목+여백 공간 확보 위해 82vh)
  작은 다이어그램은 상한보다 작아 영향을 받지 않는다.
*/
.mermaid { text-align: center; }
.mermaid svg,
.mermaid img,
p > img,
p > svg,
img,
svg {
  max-width: 100% !important;
  max-height: 82vh !important;
  height: auto !important;
}
