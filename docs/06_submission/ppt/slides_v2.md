## 1. 기획 배경 · 목표

**문제 인식**
- 여행 계획 시 관광지 검색 → 지도 → 길찾기 앱을 **여러 서비스를 오가며** 반복
- 후보 장소를 모아두고 **하루 단위로 동선을 짜는** 도구 부재
- 장소 간 **현실적 이동 시간**을 일정에 반영하기 어려움

**목표 — "한 화면에서 끝나는 여행 일정 도구"**
- 관광지 탐색 → 후보군 → **드래그앤드롭 일정 확정**을 한 흐름으로
- 구간별 **이동 시간·수단 자동 계산** 및 경로 지도 시각화
- 완성한 일정을 **커뮤니티로 공유**, 일행과 **실시간 협업**

> 추가 기능 중심: 이동시간 자동계산 · 실시간 협업 · AI 챗봇 · 방문 지도

## 3. 시장 분석 — 차별화 전략

- **통합 동선 설계**: 탐색–후보군–타임라인을 한 화면(작업실)에서 끊김 없이
- **현실적 이동시간**: ODsay(대중교통) + T Map(자동차·도보·택시요금)을 **구간별 모드 선택**으로
- **협업 우선**: 공유 링크 + 실시간 동시 편집(낙관적 락) — 경쟁 서비스 미제공 영역
- **AI 도우미**: 관광지 컨텍스트 Q&A + 반경 3km 주변 추천
- **기록의 자산화**: 여행 이야기 → 방문 지도로 시각화

## 4. 개발 결과 — 핵심 기술 (1/2)

**관광지 도메인**
- 한국관광공사 TourAPI 4.0 전국 데이터 일괄 수집 → 자체 DB 서비스
- 지역·카테고리·키워드 조회, 상세(소개·이미지·이용안내), 네이버맵 마커 연동

**일정 도메인**
- Trip → Candidate → Block 모델, vuedraggable 드래그앤드롭 타임라인
- 후보군 도시(시군구) 자동 분류, 즐겨찾기 자동 연동, 중복 방문 허용

## 4. 개발 결과 — 핵심 기술 (2/2)

## 5. 전체 시스템 구조도

```
 [Vue 3 SPA] ──REST──▶ [Spring Boot]
   Views/Pinia          Controller→Service→MyBatis ──▶ [MySQL 8.0]
   STOMP Client ◀─WS─▶  WebSocket /ws
                        Spring Batch ──▶ TourAPI
        │                Service ──▶ ODsay / T Map / gms / Kakao
        └──지도(Naver Maps)
```

- 공통 응답 `ApiResponse{success,data,message,errorCode}`
- JWT 필터 인증, 서버 측 소유권/역할 검증
- 외부 API 키는 전부 환경변수 주입

## 6. 화면 흐름도 및 시연

**흐름**: 랜딩 → (로그인/카카오) → **여행 작업실** ↔ 여행 이야기 ↔ 마이페이지

- **작업실(/plan)**: 탐색 모드(검색·후보군) ↔ 정리 모드(드래그 타임라인·이동시간·지도)
- **여행 이야기(/community)**: 목록 → 상세(좋아요·북마크·댓글) / 글쓰기(Tiptap)
- **마이페이지(/mypage)**: 여행·내정보·내장소·방문지도·내가쓴글·북마크·좋아요

> 시연 순서: 회원가입 → 관광지 탐색·후보군 → 드래그 일정 → 이동시간·지도 → 공유·협업 → AI 챗봇 → 방문 지도

## 7. 적용 패턴 및 핵심 알고리즘 (1/2)

**이동시간 캐싱**
- 키 = (좌표 route_key, 출발시각대, 이동수단 모드) → 모드별 독립 캐시
- 캐시 정밀도 1~5단계(`system_config`), 러시아워 분기 → 무료 API 쿼터 보호
- 노선 형상은 `lane_polyline` 영구 캐시

**드래그앤드롭 타임라인**
- 후보군→Day 그리드 드롭으로 블록 생성, 30분 스냅, 사이드바 삭제존
- 날짜 범위 이탈은 DB TRIGGER로 방어

---

## 7. 적용 패턴 및 핵심 알고리즘 (2/2)

**실시간 협업 — 낙관적 락**
- `trip_block.version`으로 동시 편집 충돌 감지 → 후행 저장 거부·재동기화
- transit 재계산은 version 미변경(불필요한 충돌 방지)

**AI 주변 추천**
- `ST_Distance_Sphere`로 반경 3km 거리순 조회 → 프롬프트 주입
- 응답 언급 장소만 버튼화 → 지도 핀·상세 이동, 뒤로가기 복원

**데이터 보존**
- 작성자/일정 삭제 SET NULL 보존, 글 소프트 딜리트 + 북마크 보존, 후보군 RESTRICT

## 8. 기대 효과

- **여행 준비 시간 단축**: 탐색–동선–이동시간을 한 화면에서 → 앱 전환 비용 제거
- **현실적인 일정**: 구간별 실제 이동시간·수단으로 무리 없는 동선
- **함께 만드는 여행**: 실시간 협업·공유로 일행과 합의된 일정
- **경험의 축적**: 여행 이야기·방문 지도로 기록을 자산화
- **확장성**: 도메인 중심 설계 + 캐시 전략으로 트래픽·기능 확장 용이
