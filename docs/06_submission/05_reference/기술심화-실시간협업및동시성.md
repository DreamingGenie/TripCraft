## 다룰 범위

공동 협업을 위한 ① **다양한 환경에서도 상대좌표를 계산해 정확한 위치를 가리키는 실시간 마우스 트래킹**,
② **WebSocket·presence를 활용한 편집 내용 실시간 적용**, ③ **동시 편집 충돌을 막는 동시성 제어**.

## 1. 실시간 커서 트래킹 — 상대좌표 동기화

해상도·스크롤·줌이 다른 환경 간에 절대 픽셀 좌표는 무의미하다 → **zone 기반 의미 좌표**(절대 픽셀 X)를 전송한다: 컬럼 좌측 비율(`colRatioX`), 콘텐츠 Y(`contentY`), 지도 비율(`mapRatioX/Y`), 드래그 grab 위치·오프셋(`grabRatioX`, `grabOffsetMin`). 수신측이 자기 화면 기준으로 환산해 표시한다.

- **적응형(AIMD) 전송·보간**: 한가하면 부드럽게(보간 상한까지), 혼잡하면 즉시 back off(스냅) → 랙 제거.
- **keepalive heartbeat**: idle에도 마지막 커서 위치 유지.
- **presence는 '포커스 기준'**: 화면에 있으면 유지, 떠나면 제거. stale(5초) eviction으로 유령 커서 정리.

## 2. 편집 실시간 적용 — WebSocket(STOMP) + afterCommit broadcast

사용자 편집 REST API가 트랜잭션 **커밋 후** `/topic/trip/{id}` 로 변경 이벤트(BLOCK_ADDED/MOVED/DELETED, CANDIDATE_*, TRANSIT_RECALCULATED)를 브로드캐스트한다. 커밋 전에 보내면 수신측이 즉시 재조회(별도 트랜잭션)할 때 미커밋 변경을 못 봐 '한 템포 늦게' 반영되는 문제가 생긴다 → **`afterCommit` + seq 부여(seq 순서=커밋 순서=전송 순서)** 로 해결.

## 3. 동시성 제어 — 낙관적 락 + grab + 행 직렬화

같은 블록 동시 수정은 막되, 서로 다른 블록 편집·시스템 transit 재계산은 충돌로 오인하지 않는다.

- **낙관적 락**(`trip_block.version`): 사용자 편집은 `WHERE id=? AND version=?` 조건부 UPDATE → 0행이면 409 → 프론트 재조회 후 재시도. `version`은 같은 row에만 작용해 무관한 블록 편집을 비간섭.
- **transit 분리**: 이동시간 재계산 UPDATE는 version 미변경(오탐 방지 전제). 외부 API 호출은 `afterCommit`으로 트랜잭션 밖으로.
- **grab 서버 게이트**(사전 차단): 드래그 중인 블록은 다른 사용자가 못 만지게 차단. stale/disconnect 시 자동 해제.
- **시간대 겹침 금지**: `SELECT ... FROM trip WHERE id=? FOR UPDATE`로 일정 편집을 직렬화해 race 방지.

## 4. 발표 포인트

1. 환경 독립 좌표(zone 의미 좌표)로 멀티 디바이스 커서 정합.
2. 충돌 기준을 행렬로 정의(같은 블록=하드 차단, 다른 블록=비간섭, transit=무간섭).
3. 낙관적 락(사후) + grab(사전) 2층 방어.
4. CRDT/OT는 의미 충돌을 못 없애고 ROI가 낮아 거부·재시도 정책 채택.
