# 시연 시나리오 — TripCraft (final)

> **버전**: final (2026.06)
> **목적**: 발표 시연(라이브/녹화) 대본. PPT "화면 흐름도 및 시연" 슬라이드의 원천.
> **구성**: 장면별 화면 경로 · 행동 · 기대 결과 · 시연 포인트(어필 기능).

---

## 시연 흐름 한눈에

```
랜딩 → 회원가입/로그인 → 관광지 탐색(지도·상세·AI챗봇) → 일정 작성(보관함·드래그·이동시간)
   → 커스텀 장소 → 실시간 협업·공유 → 커뮤니티 공유 → 마이페이지 → 관리자
```

준비물: 계정 2개(협업 시연용), 크롬 2창(또는 시크릿), 데모 일정용 관광지 미리 파악.

---

## 장면 1 — 진입·소개 (비로그인)
| 항목 | 내용 |
|------|------|
| 화면 | `/` 랜딩 → `/about` 소개 → `/discover` 탐색 |
| 행동 | 랜딩에서 서비스 가치 소개, 비로그인으로 관광지 탐색 진입 |
| 기대 | 로그인 없이도 탐색 가능. 로그인 사용자는 `/discover` 접근 시 작업실(`/plan`)로 자동 유도 |
| 포인트 | 공개 탐색 + 권한별 진입 동선(라우터 가드) |

## 장면 2 — 회원가입·로그인
| 항목 | 내용 |
|------|------|
| 화면 | `/auth` |
| 행동 | ① 이메일 회원가입(이메일·비번 8~64·닉네임 2~20) → 로그인 ② 카카오 소셜 로그인 |
| 기대 | 로그인 성공 시 `access_token`·`refresh_token` 쿠키 발급, 헤더가 로그인 상태로 전환 |
| 포인트 | 쿠키 기반 JWT, 카카오 OAuth, 토큰 자동 재발급 |

## 장면 3 — 관광지 탐색·즐겨찾기·AI 챗봇
| 항목 | 내용 |
|------|------|
| 화면 | `/discover` (또는 `/plan` 탐색 모드) |
| 행동 | 지역(시도·시군구)·카테고리·키워드 필터 → 목록·지도 마커 → 상세 패널(이미지·소개·이용안내) → 즐겨찾기 → **AI 챗봇 질문** |
| 기대 | Naver Maps 마커·상세 연동, 즐겨찾기 토글, 챗봇이 **해당 관광지 컨텍스트**로 답변 + **주변 3km 추천** |
| 포인트 | TourAPI 데이터, 지도 연동, **Spring AI 챗봇(컨텍스트 주입·주변 추천·환각 방지)** |

## 장면 4 — 일정 작성 (핵심)
| 항목 | 내용 |
|------|------|
| 화면 | `/plan` 작업실 |
| 행동 | ① 일정 생성(제목·기간·인원·기본 이동수단) ② 탐색 결과를 **보관함**에 담기(즐겨찾기 포함) ③ 날짜 탭에 **드래그 앤 드롭으로 블록 배치**·순서·시간·메모 |
| 기대 | Day 탭 자동 생성, 블록 배치 시 **이전 블록과의 이동 시간 자동 계산** 표시 |
| 포인트 | 보관함→타임라인 드래그, **ODsay 이동시간 자동 계산**, 시간 겹침 방지 |

## 장면 5 — 멀티모달 이동수단·경로 지도
| 항목 | 내용 |
|------|------|
| 화면 | `/plan` (이동 Pill·지도 패널) |
| 행동 | 이동 Pill 클릭 → 대중교통/자동차/도보 병렬 비교 → 옵션 선택 → "지도로 보기" |
| 기대 | 소요시간·요금·택시비 표시, 지도에 **구간별 색 경로·도보·역마커·폴리라인** |
| 포인트 | **T Map 자동차 4옵션·도보**, ODsay 경로 시각화, 다층 캐시 |

## 장면 6 — 커스텀 장소
| 항목 | 내용 |
|------|------|
| 화면 | `/plan`, `/mypage/places` |
| 행동 | Kakao Local 검색으로 TourAPI에 없는 장소 추가 → 보관함·내 장소로 재사용 |
| 기대 | 좌표 기반 후보 추가, 좌표 중복 방지, 커스텀도 이동시간 동일 파이프라인 |
| 포인트 | Kakao Local 연동, 관광지/커스텀 통합 경로 처리 |

## 장면 7 — 실시간 협업·공유 (하이라이트)
| 항목 | 내용 |
|------|------|
| 화면 | `/plan` (2창), 공유 모달 |
| 행동 | ① 협업자 초대(EDITOR/VIEWER) → **두 창에서 커서·아바타 실시간 표시**, 한쪽 블록 이동이 즉시 동기화 ② 같은 블록 동시 편집 시 **충돌 안내 후 재조회** ③ 공유 링크(VIEW/EDIT) 발급 → 비로그인 조회 → 일정 복제 |
| 기대 | STOMP 실시간 프레즌스·블록 동기화, 낙관적 락 충돌 처리, 공유 링크 동작 |
| 포인트 | **실시간 공동 편집(동시성 정책)**, 공유 링크·복제 |

## 장면 8 — 커뮤니티 공유
| 항목 | 내용 |
|------|------|
| 화면 | `/community`, `/community/write`, `/community/:id` |
| 행동 | 완성 일정을 게시글로 작성(대표 이미지) → 목록(정렬·검색) → 상세(**공유 일정 뷰어**) → 좋아요·댓글/대댓글·북마크 |
| 기대 | 게시글 CRUD(소프트 딜리트), 일정 뷰어, 상호작용 반영 |
| 포인트 | 일정 공유 게시판, 댓글·대댓글, 좋아요/북마크 |

## 장면 9 — 마이페이지
| 항목 | 내용 |
|------|------|
| 화면 | `/mypage` (trips/profile/places/map/posts/bookmarks/likes) |
| 행동 | 내 일정·프로필(이미지)·내 장소·**방문 지역 지도**·내 게시글·북마크·좋아요 탭 둘러보기 |
| 기대 | 탭별 내 데이터 조회, 방문 지역 지도 시각화 |
| 포인트 | 통합 마이페이지, 방문 지역 지도 |

## 장면 10 — 관리자 (선택)
| 항목 | 내용 |
|------|------|
| 화면 | `/admin` (ADMIN 계정) |
| 행동 | 공지 작성, 관광지 데이터 동기화(전체/부분/참조) |
| 기대 | 공지 노출(커뮤니티 사이드바 최신 5건), TourAPI 동기화 결과 |
| 포인트 | 관리자 권한 분리, 데이터 운영 |

---

## 시연 강조 3가지 (시간 부족 시 우선)
1. **일정 작성 + 이동시간 자동 계산**(장면 4~5) — 핵심 가치.
2. **실시간 협업·공유**(장면 7) — 기술 난이도·차별화.
3. **관광지 AI 챗봇**(장면 3) — 추가 기능 어필.

## 리스크·대비
- 외부 API 지연: 데모 경로는 **사전 1회 조회로 캐시 워밍**.
- 협업 시연: 네트워크 불안 대비 동일 PC 2창 우선.
- AI 챗봇: 응답 지연 대비 질문 1개는 미리 시연 확인.
